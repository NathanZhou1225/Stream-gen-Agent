# Sprint Plan — streamy 文稿生成产出打磨（v0.1.8）

## Sprint Root

- `/root/.openclaw/workspace-stream-gen`

## Goal

- 在不引入公域账号爬取的前提下，完成本轮文稿生成打磨：  
  1) 选题阶段补齐「标题 + 论点论据 + 热榜弱舆情信号」；  
  2) 制作指导实现「大纲轻提示 + 逐字稿附录详细执行」双层输出；  
  3) user-style 从「对话采集」切换为「资料驱动档案化」并稳定影响生成结果。

## Context / Constraints

- 对齐 `RECORD.md`：当前聚焦功能①内容产出，保持三段式流程 `topic_picking -> outline_refining -> script_refining`。
- 继续遵循既有约束：  
  - 选题轮展示 digest，不跳阶段直接给整稿。  
  - Draft ID 契约不变（3 字符 + `active/default/<id>/`）。  
  - 主链工具脚本仍不在脚本内直接调 LLM 成稿。  
- 本 Sprint 明确排除：  
  - 公域平台账号级爬取与评论细粒度情绪分析。  
  - 大产品功能②（内容复盘）范围扩张。  

## Deliverables

- 选题结构升级：每个候选包含标题、核心论点、论据、弱舆情字段。
- 大纲输出新增轻量制作提示段（不喧宾夺主）。
- 逐字稿附录新增详细制作指导（镜头/贴纸特效/配图/人物行为）。
- user-style 档案输入链路（网络资料 + 个人介绍 + 历史样本）与注入规则。
- 验收样例与抽检口径（事实追溯、风格一致、信息密度）。

## Task Checklist

### P0（本 Sprint 必做）

- [ ] T1. 冻结选题输出 schema（标题 + 论点 + 论据 + 弱舆情）
  - 输出字段草案（建议）：`headline`, `thesis`, `evidence[]`, `weak_sentiment{source, ts, signal, note}`
  - 约束：`evidence` 固定 3 条，且每条附 `confidence=high|medium|low`，尽量可追溯到行情/快讯/公告。
  - DoD：业务可仅看候选列表直接判断“是否值得开稿”。

- [ ] T2. 接入热榜弱舆情信号（信息侧最小可用）
  - 先做两路免费可落地来源，不做账号级爬取。
  - 输出统一标签：来源、时间、信号强度（可简化）、一句解释。
  - DoD：至少一条候选可稳定展示弱舆情字段；若两路都失败，除结构化降级外必须在飞书端输出明确文字提示（本轮基于行情/快讯/公告生成）。

- [ ] T3. 大纲阶段增加轻量制作提示
  - 每段 1 行提示，控制篇幅，不覆盖正文逻辑。
  - DoD：大纲可读性不下降，同时能看到拍摄/剪辑方向。

- [ ] T4. 逐字稿附录增加详细制作指导
  - 固定附录结构：`镜头建议`、`贴纸/特效`、`配图建议`、`人物行为`。
  - 每块建议 3-5 条，可执行、非空话。
  - DoD：主播和剪辑可按附录直接执行，无需二次口头翻译。

- [ ] T5. user-style 非对话档案化（资料驱动）
  - 输入来源：个人介绍、可用网络资料、历史样本。
  - 档案字段：`bio`, `ip_positioning`, `audience`, `tone`, `taboo`, `structure_pref`, `visual_pref`, `evidence_pref`
  - DoD：同题不同用户产出具可识别风格差异，且可复现。

- [ ] T6. 观点正确度最小守门（轻量）
  - 区分事实陈述与观点判断；论据尽量标注可追溯来源类型。
  - DoD：抽检时能明确分辨“事实/判断”，降低误导风险。

### P1（本 Sprint 有余力则做）

- [ ] T7. 弱舆情质量监控与降噪规则
  - 建立简易抽检指标：可用率、噪声率、与选题相关性。
  - DoD：可给出来源保留/降权建议。

- [ ] T8. 附录模板个性化（按 user-style 微调）
  - 让镜头/贴纸/行为建议随 IP 风格适配。
  - DoD：同主题下不同风格模板差异明显且稳定。

## Execution Rhythm (2 Weeks)

- Week 1（骨架可用）
  - D1-D2：完成 T1 schema 冻结 + T2 弱舆情最小接入设计。
  - D3-D4：完成 T3 大纲轻提示 + T4 附录结构落地。
  - D5：联调与业务评审，冻结 P0 输出格式。

- Week 2（质量提升）
  - D1-D2：完成 T5 user-style 档案化注入。
  - D3：完成 T6 观点正确度守门与展示规范。
  - D4：推进 T7（若有余力）并做回归抽检。
  - D5：整理发布候选版本与遗留项转 backlog。

## Acceptance Gate (Sprint Exit)

- [ ] 选题阶段稳定输出：标题 + 论点 + 论据 + 弱舆情信号。
- [ ] 当两路热榜源均不可用时，飞书端必须出现明确降级提示文案。
- [ ] 大纲轻提示与逐字稿详细附录并存，且不影响主文可读性。
- [ ] user-style 档案无需对话采集即可驱动个性化。
- [ ] 事实与观点可区分，关键论据具可追溯性。

## Risks / Mitigation

- 热榜噪声高：多来源对拍 + 降级输出，不强行生成高置信舆情结论。
- 信息量上升导致冗长：限制论据和附录条目上限，优先高价值信息。
- 风格资料质量不齐：字段缺失时回退默认模板并记录缺口。
- 范围膨胀：公域爬号、深舆情分析统一进 backlog，不占本 Sprint 主线。

## Out of Scope

- 公域账号爬虫体系建设。
- 舆情供应商级深度接入与评论情感细粒度模型。
- 内容复盘系统（功能②）开发。
