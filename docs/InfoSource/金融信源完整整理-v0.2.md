## 0. 系统环境与输出规范

### 0.1 核心设计理念

`finance-source-ingest` 的核心设计理念是：

> **主干无模型，提纯赖大模型。**

也就是说，主链路的抓取、清洗、归一化、降级与错误记录，全部由确定性代码完成；只有在最后的六大板块重组阶段，才把候选菜单交给 LLM Router 做信息提纯。

这样设计的原因是：

- 金融事实不能靠模型“猜”。
- 原始事实必须可审计，可在 `sections` 中追溯。
- 失败必须显式记录到 `errors`，不能静默吞掉。
- LLM 只做“选什么更值得讲、怎么归入板块”的判断，不负责制造事实。

### 0.2 当前模型底座

当前 OpenClaw 默认模型配置：

- 模型：`ark/ark-code-latest`
- 平台：火山方舟北京节点
- API 兼容：`openai-completions`
- Context Window：`200000`
- Max Tokens：`8192`
- Agent 默认超时：`240s`

注意：

- `finance-source-ingest` 的抓取主链路不调用 LLM。
- `sections.llm_router` 会使用模型做板块候选重组。
- P1 小 LLM 润色灰度当前已证明 `ark-code-latest` 对多板块小请求仍偏慢，默认应保持关闭或走规则回退。

### 0.3 JSON 快照结构

当前快照主要包含：

- `sections.market`
- `sections.macro_hot`
- `sections.global_macro`
- `sections.news`
- `sections.social`
- `sections.deep_news`
- `sections.llm_router`
- `errors`
- `markdown_summary`
- `meta`
- `invariants`

其中：

- `sections` 是机器事实层。
- `errors` 是失败与降级审计层。
- `markdown_summary` 是面向飞书/用户展示的阅读层。
- `sections.deep_news` 保留原始深度内容；深度内容不再单列为展示小节，而是由 Router 拆解并入六大板块候选池。
- `sections.llm_router` 记录 Router 的状态、选择结果、诊断与耗时。

### 0.4 展示结构

当前 `markdown_summary` 的用户可见结构是：

1. 大盘与情绪
2. 六大板块异动
3. 大事件
4. 今日热点讯息
5. 全球宏观
6. 社媒 / 人气榜
7. 今日值得开稿 Top 3
8. 中文告警

## 1. 信源总览

### 1.1 主干信源

主干信源是系统最依赖、最不希望失败的一层。

| 信源 | 当前角色 | 主要提供 |
|------|----------|----------|
| 新浪财经 `hq.sinajs.cn` | 行情底座 | A 股三大指数、港股核心指数、可选海外/大宗 |
| RSSHub 本地服务 | 资讯网关 | 快讯、深度内容、六大板块垂直路由 |
| 新浪财经 7x24 | 宏观实时源 | 宏观快讯、海外市场、北向文本降级 |
| 证监会 / 央行官网 | 权威政策源 | 监管公告、政策事件、金融合规背景 |
| Tushare | 资金面增强 | 北向资金、可选 CCTV 新闻联播 |

### 1.2 增强信源

增强信源有价值，但不允许拖垮主链路。

| 信源 | 当前角色 | 主要提供 |
|------|----------|----------|
| AkShare | 增强探测层 | 行业资金、涨跌停、东财人气、财联社补源 |
| 财联社（AkShare 封装） | 快讯补源 | 财联社重点/全部快讯，双源共振 |
| 百度热榜 | 热点补位 | 泛财经热点、社媒兜底 |
| vvhan 微博热搜 | 社媒首选但不稳定 | 微博热搜标题与热度 |

### 1.3 回退/隔离信源

| 信源 | 当前状态 | 原因 |
|------|----------|------|
| 直连华尔街见闻 API | 保留回退 | 可能空返回 |
| 直连第一财经 RSS | 低可靠 | 404 风险高 |
| 直连金十 RSS | 低可靠 | DNS/出网波动 |
| 直连界面 RSS | 低可靠 | 404 风险高 |
| Tavily / WebSearch | 默认链路隔离 | 不能混入事实 JSON |
| AkShare 淘股吧/问财函数 | 实验/失败源 | 当前函数常缺失 |

## 2. 大盘与资金面（Market & Fundamentals）

### 2.1 业务目标

大盘与资金面是整套快照的硬核事实底座，用来回答：

- 今天市场涨跌如何？
- 哪些行业强？
- 北向资金如何？
- 主力资金往哪里流？
- 有没有明显情绪热词或热门个股？

### 2.2 驱动模块

- `fetchers/market.py`

### 2.3 信源与数据产出

#### 新浪财经 hq：绝对主干

当前主路径：

- A 股三大指数：
  - 上证指数
  - 深证成指
  - 创业板指
- 港股核心指数：
  - 恒生指数
  - 恒生科技指数
  - 恒生中国企业指数
- 可选海外/大宗：
  - 道琼斯
  - 标普 500
  - 纳斯达克
  - VXX
  - COMEX 黄金
  - WTI 原油

稳定性：

- A 股/港股指数：高
- 海外/大宗：中，且默认不开，需要 `FINANCE_SOURCE_OVERSEAS_STUB=1`

为什么它是主干：

- 不需要 token。
- 接口轻量。
- 云服务器访问比东财链路友好。
- 即使 AkShare 被 WAF 或超时，指数仍能稳定输出。

常见错误：

- `SINA_TRINITY_FAILED`
- `SINA_TRINITY_EMPTY`
- `SINA_HK_INDICES_FAILED`
- `SINA_HQ_FAILED`

兜底策略：

- 指数失败时保留占位结构，避免下游 JSON 解析中断。
- 不从 `markdown_summary` 中补造数值。

#### AkShare：增强探测层

AkShare 在大盘与资金面中负责增强字段：

- `stock_hsgt_fund_flow_summary_em`：北向资金主探测
- `stock_fund_flow_industry`：行业涨跌与主力净流入
- `stock_zt_pool_em`：涨停池
- `stock_dt_pool_em`：跌停池
- `stock_hot_tgb`：淘股吧热榜，当前常失败
- `stock_hot_keyword_em`：东财热词
- `stock_hot_rank_wc`：问财热榜，当前常失败
- `stock_hot_rank_em`：东财人气榜

稳定性分层：

- 相对可用：`stock_fund_flow_industry`、`stock_hot_rank_em`
- 不稳定：北向、涨跌停池、东财热词
- 常失败：`stock_hot_tgb`、`stock_hot_rank_wc`

环境开关：

- `FINANCE_SOURCE_SKIP_AKSHARE_PROBE=1`

启用该开关后：

- 保留新浪指数主链路。
- 跳过北向/行业/涨跌停/情绪等 AkShare 扩展探测。
- 适合火山 ECS 遇到东财 WAF 严重时使用。

常见错误：

- `AKSHARE_IMPORT_ERROR`
- `AKSHARE_PROBE_SKIPPED`
- `NORTHBOUND_PROBE_FAILED`
- `INDUSTRY_RANK_PROBE_FAILED`
- `TEMPERATURE_POOL_PROBE_FAILED`
- `INFLOW_SECTOR_PROBE_FAILED`
- `TGB_HOT_FAILED`
- `WC_RANK_FAILED`
- `EM_KEYWORDS_FAILED`
- `EM_RANK_FAILED`

#### 北向资金四级降级链

北向资金是当前最典型的多级容错字段。

优先级：

1. AkShare `stock_hsgt_fund_flow_summary_em`
2. Tushare `moneyflow_hsgt`
3. 新浪 7x24 文本正则抽取
4. RSSHub `/wallstreetcn/live` 文本抽取

这么设计的原因：

- AkShare 北向接口交易时段口径容易波动。
- Tushare 需要 token，但字段更结构化。
- 新浪 7x24 和 RSSHub 文本只能算估算/提示，但能避免字段完全空。

## 3. 六大垂直产业板块（Sector Heterogeneity）

### 3.1 业务目标

六大板块是面向金融自媒体开稿最重要的内容区。目标不是简单列新闻，而是输出可讲的结构化材料：

- 事件：发生了什么
- 影响：为什么重要
- 角度：如何开稿/怎么讲
- 来源：来自哪个信源
- 情绪：利好、利空、中性

当前六大板块：

- 科技
- 新能源
- 港股
- 黄金
- 有色
- 银行

### 3.6 Ark LLM Router 提纯

Router 输入：

- `sections.news`
- `sections.global_macro`
- `sections.deep_news`
- `sections.macro_hot`
- `items_other_flash`
- 已有板块桶

Router 目标：

- 每个板块精选 2-4 条。
- 同时覆盖盘面快讯和产业/基本面源。
- 避免纯时间排序把深度源挤掉。
- 过滤弱相关、重复、泛科技噪声。

成功状态：

- `ok`
- `ok_compact_retry`

降级状态：

- `fallback_grouped`
- `disabled_by_env`
- `no_candidates`

失败错误：

- `LLM_ROUTER_TIMEOUT`
- `LLM_ROUTER_FAILED`

兜底：

- Router 超时/异常时自动走 `fallback_grouped`。
- 不阻断 JSON 输出。
- 降级后仍尽量按板块强关键词分组。

### 4.4 今日热点讯息

今日热点讯息不是单一信源，而是 pipeline 从全源候选池中过滤：

- `global_macro`
- `deep_news`
- `news.items_other_flash`
- `macro_hot`
- 非六大板块但金融相关的新闻

过滤目标：

- 不与六大板块重复。
- 不与大事件重复。
- 保留全局影响力强但不归属某一板块的内容。

## 5. 社媒与舆情（Social Sentiment）

### 5.1 业务目标

社媒/舆情层用于捕捉：

- 散户关注度
- 热门个股
- 弱舆情方向
- 可能值得做短视频切入的市场热度

它不提供硬事实底座，只作为软性补充。

### 5.2 驱动模块

- `social_api.py`

### 5.3 三级降级漏斗

#### 第一梯队：vvhan 微博热搜

入口：

- `https://api.vvhan.com/api/hotlist/wbHot`

提供：

- 微博热搜标题
- 热度
- 链接

当前问题：

- DNS/出网失败风险较高。
- 金融命中率不稳定。

常见错误：

- `SOCIAL_WB_HOT_FAILED`
- `SOCIAL_WB_HOT_PARSE`
- `SOCIAL_WB_HOT_EMPTY`
- `SOCIAL_WB_HOT_NO_MACRO_MATCH`

#### 第二梯队：AkShare / 东财人气

入口：

- `stock_hot_tgb`
- `stock_hot_rank_em`

提供：

- 热门股票
- 当前排名
- 最新价
- 涨跌幅

当前问题：

- `stock_hot_tgb` 高频不可用。
- `stock_hot_rank_em` 相对可用，但只能告诉“谁热”，不能解释“为什么热”。

常见错误：

- `SOCIAL_TGB_OR_HOT_RANK_FAILED`
- `SOCIAL_TIER2_EMPTY`
- `SOCIAL_USED_TIER2_FALLBACK`

#### 第三梯队：百度实时热榜

入口：

- `https://top.baidu.com/board?tab=realtime`

提供：

- 百度实时热词
- 经过财经关键词过滤后的热点

策略：

- 只输出命中金融/宏观关键词的内容。

### 5.4 当前短板

社媒层最大问题仍然是：

## 6. 隔离区与失败池（The Dead Pool）

### 6.1 AkShare 隔离项

已降级或视为实验源：

- `stock_hot_tgb`
- `stock_hot_rank_wc`

原因：

- 当前 AkShare 版本常无这些函数。
- 会产生噪音错误。
- 不应作为主链路依赖。

### 6.2 媒体直连源

直连源包括：

- 华尔街见闻 API
- 第一财经 RSS
- 金十 RSS
- 界面 RSS

问题：

- 404
- DNS/出网失败
- 空返回
- 上游结构变化

当前策略：

- RSSHub 优先。
- 直连仅作回退。
- 直连失败写入 `errors`，不阻断主链路。

### 6.3 Tavily / WebSearch

状态：

- 保留为独立 skill。
- 不进入默认 `finance-source-ingest` JSON。

原因：

- 搜索结果不能覆盖 API 数值事实。
- 过去试过 `meta.websearch_required` / `meta.websearch_gaps`，但 Agent 未必真的搜索。
- 混入搜索附录会让事实边界不清。

当前规则：

- 缺口写 `errors`。
- 用户追问时 Agent 可自愿联网说明。
- 不得把联网结果伪装成 ingest 原始信源。

当前架构已经具备热插拔基础：各信源都被隔离在 fetcher 层，输出统一进入 `sections` 与 `errors`。未来商业级升级不需要推翻架构，只需要替换或增强 fetcher。

- 降低 WAF 风险。

## 10. 当前最关键的维护原则

1. **新浪 hq 继续做行情主干**。不要让 AkShare 重新成为指数唯一主源。
2. **RSSHub 必须进程守护**。它是资讯层当前最重要的稳定性工程。
3. **AkShare 只做增强层**。失败可见，但不能拖垮主链路。
4. **Tavily/WebSearch 不进入事实 JSON**。搜索只能解释缺口，不能替代 API 事实。
5. **Router 可以失败，但快照不能失败**。`fallback_grouped` 是必须保留的安全底线。
6. **每条失败必须写 errors**。没有错误码，就无法排障。
7. **面向飞书的展示要固定结构**。不要让 Agent 根据用户措辞临场改变快照形态。

## 11. 一句话总结

当前 `finance-source-ingest` 已经形成：

> **新浪行情打底 + RSSHub 内容增厚 + AkShare/Tushare 补资金和人气 + 政策官网保权威 + 百度/微博补热度 + Ark Router 做板块提纯 + Top3 做开稿引导。**

其中真正的主依赖是 **新浪 hq、RSSHub、新浪 7x24、政策官网、Tushare**；AkShare 继续有价值，但必须保持增强层定位；RSSHub 当前承担了快讯、深度和六大板块垂直路由三类核心任务。
